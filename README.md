CS‑330 Final Project Reflection
How I Approach Designing Software
My approach to software design starts with breaking a problem into smaller, understandable pieces. I focus on identifying the core requirements, then mapping out how each part of the system should behave before I ever write code. Working on this project reinforced the value of planning the structure of a program early, especially when dealing with graphics, where objects, transformations, textures, and user interaction all depend on clear organization.

New Design Skills I Developed
Building the 3D scene helped me strengthen modular design and scene organization. I learned how to design reusable components, such as object classes, transformation functions, and texture loaders. I also gained a better understanding of how lighting, materials, and camera controls fit together as part of a cohesive design.

Design Process I Followed
My design process was iterative:

I started with simple primitives

Added textures and materials

Introduced lighting and shading

Refined camera movement and user interaction

Each step built on the previous one.

How These Tactics Apply to Future Work
Breaking problems down, designing modular components, and iterating applies to any future software project.

How I Approach Developing Programs
I focus on building a working foundation first, then layering complexity on top of it. For this project, that meant getting the basic rendering pipeline working before adding textures, lighting, and movement.

New Development Strategies I Used
Incremental rendering

Shader experimentation

Texture debugging

How Iteration Factored Into Development
Iteration was essential. Each milestone required revisiting earlier work — improving object placement, adjusting lighting, reorganizing code, and refining textures.

How My Development Approach Evolved
At the beginning, I focused mostly on getting things to work. By the end, I was thinking more about structure, readability, and scalability.

How Computer Science Helps Me Reach My Goals
Computer science gives me tools to solve problems, automate tasks, and build systems that can scale.

How Computational Graphics Helps My Educational Path
Graphics and visualization give me a deeper understanding of how math, programming, and hardware interact.

How Computational Graphics Helps My Professional Path
These skills open doors in game development, simulation, virtual environments, scientific visualization, and technical software engineering.
